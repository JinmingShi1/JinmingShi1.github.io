自建服务器上的多 Agent 学习系统

Autonomous agents · 自研调度 · 生产运行中
● 每天两班运行

在一台自管的云服务器（ARM / Ubuntu）上跑着两个领域 Agent（科技 & 金融）， 每天早晚各一班：自动汇聚当日信源 → 融合实时行情与宏观指标 → LLM 做深度研究与研判 → 生成结构化日报 → 分发到多个渠道。信源中枢、行情快照、报告归档全部自动衔接，无人值守。

RSS 聚合→行情/宏观 API→拼装上下文→LLM Agent 研究→结构化日报→Discord + Notion 归档
Cloud VM (ARM) cron 调度 DeepSeek LLM Miniflux RSS Finnhub / CoinGecko / FRED Discord API Notion API Docker
🔒 私有 —— 涉及个人信源与账号，仅展示架构

A股每日自动分析

GitHub Actions · 定时量化 · 开源
● 定时任务

一条完全跑在 GitHub Actions 上的免费流水线：收盘后定时触发，对一篮子龙头股逐只抓取行情、 做技术面分析并用 LLM 生成解读，最后汇总推送。含对推送层的多处稳定性改造 （限流重试、长报告分块与附件化投递），保证长期无人值守也不刷屏、不丢消息。

Actions cron→逐股行情抓取→技术面 + LLM 解读→汇总记分板→Discord 推送
GitHub Actions Python DeepSeek LLM efinance / 腾讯行情 Discord Webhook

n8n 自动化工作流

低代码编排 · 自托管
● 每日运行

自托管 n8n 上编排的一组工作流。代表作是「法语学习助手」：每天拉取法语新闻播客的最新一集 → 下载音频作为附件推送（可在客户端直接边听边看）→ 抓取全文转写 → LLM 生成中文精读笔记 → 自动分条投递。另有服务健康监控，只在依赖服务异常时才告警。

播客 RSS→下载音频→抓全文转写→LLM 精读→分条私信推送
n8n (self-hosted) RSS / HTTP Gemini LLM 二进制/附件处理 Discord API
🔒 自托管实例 —— 仅展示工作流设计

双语字幕 Chrome 插件

浏览器扩展 · 实时翻译

给流媒体加「原文 + 中文」双语字幕的 Chrome 扩展。技术核心是在页面主世界劫持 fetch/XHR 抓取播放器加载的字幕文件（VTT / TTML / 原生文本轨多路兜底），解析后按行号批量调 LLM 翻译， 再绘制随时间轴精确同步的双语覆盖层。用户填自己的 API Key，隐私自控。

劫持字幕请求→多格式解析→按行批量翻译→时间轴同步覆盖层
Chrome Extension (MV3) JavaScript fetch/XHR 劫持 VTT / TTML 解析 DeepSeek API
🔒 个人使用 —— 可按需开源

量化回测引擎

策略研究 · Python

自己写的一套轻量回测框架：数据获取与本地缓存、策略定义、回测执行与绩效可视化（资金曲线）分层解耦。 用来快速验证交易策略想法，把「一个念头」变成可量化、可比较的结果。

行情数据 + 缓存→策略信号→回测撮合→绩效 & 资金曲线
Python pandas matplotlib 数据缓存层
🔒 个人研究 —— 可按需开源

我擅长什么

跨过「能跑一次」和「每天稳定跑半年」之间的那道坎。

把 LLM 接进真实系统

不止调 API —— 上下文拼装、多源信息融合、成本/限流治理、输出结构化、幻觉与来源约束。

端到端自己扛

从选服务器、装服务、写调度，到数据管道、分发渠道、监控告警，一个人打通全链路。

为「长期运行」而做

重试、去重、限流退避、幂等、脱敏 —— 让系统在无人值守下也不炸、不刷屏、不丢数据。

低成本落地

善用免费额度与开源工具（Actions / 自托管 / 免费行情源）把想法用近乎零成本跑起来。

关于隐私：这些系统大多在处理我个人的信源、账号与数据，所以标为「私有」。 这里展示的是架构、技术选型和工程取舍 —— 需要更深入了解某个项目的实现细节，欢迎单独聊。
